01 · 背景

叫叫出海团队，
负责 JoJo Math增长链路

叫叫（成都书声科技）是国内头部儿童成长数字内容平台，深耕儿童阅读领域，拥有"小鸡叫叫"系列 IP 与叫叫阅读课等核心产品，员工规模超 6000 人。JoJo Math 是叫叫旗下面向海外市场的儿童数学学习产品，在台湾 → 北美（美国 / 加拿大）等地运营。

公司背景
叫叫（成都书声科技），2018 年成立，高新技术企业，员工 6000+ 人，深耕儿童成长数字内容——阅读、数学、语言、思维多方向布局，是国内儿童教育领域的重要平台。
我的角色
产品实习生（出海方向），独立负责 JoJo Math 的 Push 推送系统、用户激励增长（邀请码系统）、手机号注册登录三个核心 P0 模块，并参与原生订阅与支付体系（iOS/Android IAP + Web 支付），从 PRD 到联调上线全程跟进。
核心挑战
儿童教育产品的出海场景比通用 App 更复杂——用户是家长，决策链更长；内容需与本地教学体系匹配；多国合规（号段格式、数据隐私、推送权限）要求不一；时区差异影响推送时机与用户行为分析。任何模块都不能照搬国内经验，要在约束中找到最小可用方案，快速上线验证。

01
App Push 推送系统
JoJo Math · FCM + Lark · 多语言 & 时区适配
从 0 到 1 ›
做了什么 & 关键决策

主导构建 JoJo Math 海外 App 推送体系。初期技术方案采用静态批量推送，但运营团队要求能基于用户实时状态（当日是否已练习、距上次登录天数、诊断进度）做个性化推送——原方案无法支持。

与后端协商后，提出折中方案：由后端在每日凌晨批量同步用户状态快照至推送系统，以「准实时」替代「真实时」——在几乎不增加开发成本的前提下，支持按用户状态分层推送。同时为后续真正的事件驱动推送预留了接口扩展路径。方案获得技术与运营双侧认可，按期上线。

同步解决多语言文案管理（繁中/英/西班牙语分版本）与时区推送时机适配（北美 PST/EST、台湾 CST 分组策略）。

FCMLark BotA/B Test 多语言文案时区适配用户状态同步
上线成果与逻辑链
  • 推送触达率 95%+，A/B 测试稳定值
  • 推送点击率 23%（行业均值 ≈ 5–10%）
  • 注册率提升 +15% — 对「已下载未完成注册」用户定向推送，带动注册转化
  • 数学数学诊断完成率提升 +8% — 注册完成后触发诊断引导 Push，提升用户进入核心学习路径的比例
02
用户激励增长 · 邀请码系统
注册兑换体验、权益控制、后台全生命周期管理
从 0 到 1 ›
做了什么

设计并实现注册流程中的邀请码兑换体验、权益控制与人群限制校验机制。搭建后台支持邀请码全生命周期管理（创建 / 配置 / 追踪），与后端配合实现试用期动态发放（兑换成功后 +21 天体验期）、使用限制校验及数据同步至投放追踪平台，支持拉新归因与活动转化追踪。

后台邀请码表设计涵盖：活动名称（campaign_name）、有效期、总限量 / 个人限量、目标人群（新用户 / 全部 / 老用户）、权益天数等核心字段，支持运营按场景灵活配置，兑换逻辑在客户端→后端→Lark 三层完成校验与记录同步。

拉新归因活动转化追踪 权益控制人群限制校验 后台管理试用期动态发放
上线成果
  • 在约 4000 用户基数下，邀请码渠道带来新用户增长 +15%（约 600 名新用户）
  • 拉新归因全链路打通，支持区分自然增长与邀请渠道
  • 后台可视化追踪每批邀请码的转化效果
03
原生订阅与支付体系 · P0（参与）
iOS/Android IAP + Web 支付 · 32 国上线 · 完整变现闭环
参与 ›
做了什么 & 关键决策

参与 JoJo Math 出海订阅变现体系的 PRD 与落地——覆盖 iOS（App Store IAP）与 Android（Google Play IAP）原生订阅，以及 Web 端第三方支付（Airwallex）两套路径。产品形态为 21 天免费试用 + 每 3 个月自动续费，台湾等特定地区独立定价（NT$1,800 / 季）。

核心复杂度在于：平台差异巨大——Apple 有 16 天宽限期不可取消、Google 支持暂停订阅（最长 3 个月）、Web 端需后端 Cron 任务主动轮询续费，三套机制必须在产品层面收敛为统一的权益状态模型与 UI 逻辑。设计了「正式学员 / 体验学员 / 非学员」三档用户状态，驱动首页角标、家长中心、强弹引导（benefit_days ≤ 1/3/7/14 四档触发）的差异化策略。

退款流程、取消订阅、跨平台恢复购买、自动续费失败宽限期处理均完整覆盖，并提供 Apollo 远程配置方案（台湾 21 天 / 其他地区 3 天试用）支持灰度上线。

iOS IAPGoogle Play IAPAirwallex Apollo 远程配置权益状态机退款 / 恢复购买 多语言 UI（TC/EN/ES/PT）
上线范围与核心交付
  • 覆盖 32 个国家 / 地区同步上线，含美、英、加、日、德、法、台、港、澳等主要市场
  • 完整 PRD 覆盖 8 种订阅场景（首订 / 续费 / 取消 / 暂停 / 恢复 / 退款 / 宽限期 / 跨平台）
  • 四语言（繁中 / 英 / 西班牙语 / 葡语）UI 文案表、日期格式规范与隐私政策更新全部交付
  • 正式 SKU（Apple ID 6747865172）与测试 SKU 联调通过，Apollo 灰度配置按期落地
04
手机号验证码注册与登录
多国合规调研、PRD 撰写、完整流程设计
独立交付 ›
做了什么

独立负责需求分析与全链路交付。调研不同国家手机号段格式与合规要求，适配多语言提示与跨时区验证码稳定性方案，完成竞品调研、PRD 撰写与完整流程设计，为三方登录扩展与权限管理奠定基础，全面支持产品出海场景。

核心设计要点：带国家码 8–15 位 / 不带国家码 7–11 位的动态校验规则、4 位验证码自动提交、60 秒倒计时冷却、发送前 CAPTCHA 安全验证防刷，以及新用户自动初始化多语言用户名（JOJO小学员 / Little Learner JO / Pequeño Aprendiz JO 等）。

多国合规调研PRD 撰写 竞品分析流程设计 多语言提示跨时区稳定性
核心价值
  • 覆盖多国合规，支持出海登录场景
  • 为三方登录扩展预留接口
  • 完整 PRD 与流程文档交付
  • 跨时区验证码稳定性方案落地

用户测试 & 儿童访谈 · Joen MVP
叫叫新产品 Joen（背单词）· MVP 阶段 · 儿童用户测试
用户研究 ›
做了什么

参与叫叫新产品 Joen（儿童英语背单词 App）的 MVP 用户测试全流程。负责接待真实儿童用户（及陪同家长），主导访谈设计与现场执行——引导孩子自然使用产品、同步观察操作路径与情绪反应，访谈结束后与家长深度沟通使用体验、付费意愿与对功能的期望。测试结论汇入产品迭代优先级决策。

儿童用户测试 家长访谈 MVP 验证 行为观察 访谈设计 Joen · 背单词
核心收获
  • 直接与低龄用户（和家长）接触，建立对「儿童 + 家长双角色」产品决策链的认知
  • 学会观察儿童用户的非语言信号（注意力流失、重复操作、面部表情），弥补儿童无法精准表达的访谈局限
  • 掌握如何在不引导答案的前提下，从家长端提取真实付费动机
  • 测试结论直接影响 Joen MVP 的功能优先级排序

04 · 成长与反思

在真实战场里，
建立产品判断力

不是实验室里的练习题，每一个决策都要对上线后的数据负责。

🌍
出海视野
从台湾出发，延伸至北美乃至全球 32 个市场——时区、语言、合规要求、平台规则、家长行为习惯，每个市场都有截然不同的约束。第一次从产品设计层面直面国际化复杂度：PST/EST 双时区推送分组、北美 COPPA 儿童隐私合规、Apple / Google 订阅平台差异、繁中/英/西/葡多套文案管理。学会在约束里找最优解。
💳
增长与变现的完整视角
订阅系统让我第一次完整经历「变现产品」的设计复杂度——不只是界面，而是权益状态机、平台差异抹平、退款流程、宽限期策略、多平台恢复购买。意识到增长（Push / 邀请码）和变现（订阅）必须协同设计：试用期触达率直接影响转化漏斗的起点。
📊
数据驱动决策
Push 系统上线后实时监控触达率与点击率，用 A/B 测试量化每个设计选择的真实效果。不凭感觉说话，数字才是最有说服力的产品语言。
🔗
全链路思维
从需求调研、PRD 撰写、流程设计到后端联调、上线监控，独立跑通完整链路。建立了「产品不只是原型」的认知——交付是起点，数据是终点。
🧒
儿童用户研究的特殊性
Joen MVP 测试让我意识到：儿童无法精确表达感受，产品判断必须来自行为观察，而非问卷。家长才是付费决策者，他们的顾虑和期待才是真正的产品需求入口。这改变了我对「用户访谈」的理解——不同用户群需要完全不同的研究方法。
🚀
创业团队节奏
在资源有限的早期团队里，学会快速判断优先级、接受不完美但可迭代的方案。速度和质量都是产品力的一部分，不能只要一个。
